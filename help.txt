Evaluate the floating point EXPRESSION.
Examples: %s -D 2 1.1 + 1.2
          %s -I "sqrt(2) + prod(1,2,3,4)"

Options:
  Precision:
    These options determine which values are considered equal
    -E, --epsilon=EPS    set epsilon value (default DBL_EPSILON, ~2.22e-16);
                         mutually exclusive with -T
    -T, --tolerance=TOL  set tolerance in terms of the number of double-
                         precision values between the compared numbers;
                         mutually exclusive with -E
  
  Output:
    -D, --digits=NUM     round the output to the specified number of digits
                         after the decimal point (rounds to even)
                         mutually exclusive with -I
    -I, --integer        rounds the output to an integer;
                         equivalent to -D 0;
                         mutually exclusive with -D
  
  Debug:
    --tokenize           display the output of the tokenization stage
    --postfix            display the EXPRESSION in postfix notation

  Miscellaneous:
    -H, --help           display this help text and exit
    -V, --version        display version information and exit

Operators (Standard C-precedence):
  + - * / % > < == != >= <= && ||

Functions:
  Standard:
    abs()   sqrt()  exp()   floor() ceil()  round() pow(base, exponent)
  Trigonometric:
    sin()   cos()
  Logarithms:
    ln()    natural logarithm (base e)
    log()   common logarithm (base 10)
  Aggregates (Accepts 1 or more arguments, e.g., fn(a, b, c)):
    min(...)  max(...)  sum(...)  prod(...)
  Logical Aggregates (Evaluates truthiness based on EPS/TOL):
    any(...)  all(...)
